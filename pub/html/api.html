TREE JS API

To use TreeJS API, include these two files in your html head element preceeding scripts which use TreeJS


    <script type="text/javascript" src="https://treejs309.herokuapp.com/pub/Tree.js">
    <link rel="stylesheet" href="https://treejs309.herokuapp.com/pub/Tree.css">
TreeJS

global object containing top level library methods, property of the window object

Property Type Description
Tree class main object of TreeJS library used to represent a Tree (see Tree)
TreeNode class object representing the nodes of an instantiated Tree (see Node)
setOptionsDefault function used to set properties of the default options object that sets object options that are not specified
parameters: (options)

@param options: (Object) object containing the options to set

Note: default options start as:
                                
    { 
        display: 'vertical-tree',
        contentBackgroundColor: 'darkorange',
        scale: 1,
        globalCallback: null
    }
addTreeAsChild function Creates a Tree's dom representation if not already created and adds a Tree object's dom representation to the dom
parameters: (tree, parent)

@param tree: The Tree object to append to the DOM
@param parent: the parent / containing element that the Tree's Dom representation will become a child of
Tree

Property Type Description
constructor function instantiate a Tree object
parameters: (rootid, structure, options)

@param rootid: String used as the id for the root container that will be created to hold
@param structure: structure object
@param options: Object holding all options for this Tree (see options object)

root function returns the ith root node of this Tree

@param i: integer greater than 0, used to index the root node's of the tree
collapse function collapse the entire tree so that only the root nodes are showing

sets the display css property of all children's dom elements to "none"
expand function expands the root nodes so that their children are showing

sets the display css property of all children to their previous display styles before they were set to "none" on a collapse call
expandAll function recursivley expands all nodes of the Tree so the entire Tree's nodes are visible
structure Object deep copy of the structure used in constructing this Tree object (see structure)
domNode Element div element containing this Tree's dom representation, id of this div is set to the id passed into the constructor
TreeNode

a node in a Tree Object

Property Type Description
structure Object strucuture of the tree rooted at the node (see structure)
id String id of the div containing this node's DOM representation
options Object passed down from parent, determines the various options associated with a Tree (see options)
domNode Element div containing this node's DOM representation
_visible boolean private variable used for collapsing and expanding nodes.
Modifying may have minor unintended effects within the node's DOM representation
content String or Element the content of this node in the DOM, strings are translated into span elements in the DOM, otherwise if content is type Element, then content will be directly displayed in the DOM under this node's content container (see css section)
collapse function collapses this node so none of its children are visible in the DOM
parameters: (void)
expand function expands this node so all its children are visible in the DOM
parameters: (void)
expandAll function expands this node and recursivley expands all this nodes children
parameters: (void)
removeChild function removes a child from this nodes children
parameters: (child)

@Param child: (Node) the child to remove
addChild function add a childt to this nodes children
parameters: (child, index)

@Param child: (Node) the child to remove
@Param index: (integer optional) index to insert child into this node's list of children, if not provided, child will be serted aas the last element
structure

user defined object used in constructor of a Tree or TreeNode, note that its definition is recursive

Property Type Description
<nodeid> Object the key for this property is user defined. The name of this key will be the id of the resulting node created from its value, value is an Object following the node structure specification (see node structure), there may be many of this kind of property.
node structure

user defined object used in constructor of a Tree or TreeNode, note that its definition is recursive

Property Type Description
content String or Element the content of this node in the DOM, strings are translated into span elements in the DOM, otherwise if content is type Element, then content will be directly displayed in the DOM under this node's content container (see css section)
callback function the function that will be called as the onclick for this node's content container
color String color that this node will be colored, overrides the global option
children Object each key in this object is a nodeid with a node structure as its value, represents one of this nodes children.
options

user defined object passed to constructor of a Tree

Property Type Description
display String determines how to display the tree (see examples), accepted options are:

"vertical-tree" (default): displays the tree root at the top and children branch left to right underneath

"horiztonal-tree": displays the tree root on the left and children branch top to bottom to the right

"vertical-navigation": displays the tree on the left of its container with the root at the top and each child as a list beneath it

Note: this value will be added to the class list of each TreeJS-created element in the tree, if the display contains "navigation" each node will automatically collapse or expand when clicked
contentBackgroundColor String accepts any css color, will apply as the background color to each node's content container
scale float accepted values are floats between 0 and 1, this defines the ratio of size between each child to its parent (see examples)
globalCallback function function to set as the callback for all nodes on a 'click' event unless specified in the structure Object
CSS

each part of a Tree's DOM representation can be css selected by class

class Element type Description
TreeJSRootNode div The outermost div of the tree, its content contains the rest of the Tree

parent: assigned by user upon calling TreeJS.addTreeAsChild (see TreeJS)
TreeJSNode div DOM representation of a TreeNode, contains children div.TreeJSContentContainer and div.TreeJSChildContainer

parent: div.TreeJSChildContainer
TreeJSNodeContentContainer div container for the assigned content of this node, contains child Element.TreeJSNodeContent

parent: div.TreeJSNode
TreeJSNodeContent div the assigned content of this node

parent: div.TreeJSContentContainer
TreeJSChildrenContainer div container for all the children of this node

parent: div.TreeJSNode
